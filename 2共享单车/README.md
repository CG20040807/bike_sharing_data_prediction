项目简介
本项目是共享单车骑行量预测系统，基于实际的共享单车使用数据进行分析和建模。通过对不同时间维度（季节、工作日、天气等）的骑行数据分析，采用多种机器学习算法对共享单车的日均骑行量进行预测，为单车运营提供数据支持。

核心功能
数据分析与可视化：从多个维度分析骑行量的变化规律，包括季节性、工作日效应、天气影响等
特征工程：提取日期、季节、天气、温度等多维特征
多模型对比：采用线性回归、决策树、支持向量机等三种算法进行建模
性能评估：使用MSE和R²等指标进行模型效果评估
预测分析：对测试数据进行预测并可视化对比

使用说明
环装置
确保已安装Python 3.6及以上版本
安装依赖包：
pip install -r requirements.txt

运行程序
在项目目录下运行：
python "2 共享单车骑行量预测.py"

程序将自动执行以下操作：
1.加载day.csv和hour.csv数据文件
2.进行数据预处理（时间格式转换等）
3.生成六种可视化图表
4.训练三种预测模型
5.评估各模型性能
6.输出详细的模型对比结果

输出结果
程序生成以下输出文件：
season_boxplot.png：按季节分类的骑行量箱线图
workingday_boxplot.png：工作日与非工作日的骑行量对比
weather_boxplot.png：不同天气条件下的骑行量分布
hourly_trend.png：按小时的骑行量变化趋势
temp_scatter.png：温度与骑行量的相关性散点图
monthly_avg.png：各月份的平均骑行量柱状图
model_evaluation.png：综合模型性能评估（包含R²、MSE、预测曲线、预测分布）

控制台输出详细的模型性能指标（训练/测试MSE和R²）

数据说明
每日聚合数据（day.csv）
字段说明：
instant：记录索引
dteday：日期
season：季节（1=春,2=夏,3=秋,4=冬）
yr：年份（0=2017,1=2018）
mnth：月份（1-12）
holiday：是否假期（0=非假期,1=假期）
weekday：星期几（0-6）
workingday：是否工作日（0=非工作日,1=工作日）
weathersit：天气条件（1=晴朗,2=雾,3=小雨,4=大雨）
temp：归一化温度（0-1）
atemp：体感温度（0-1）
hum：归一化湿度（0-1）
windspeed：风速（0-1）
casual：非注册用户骑行量
registered：注册用户骑行量
cnt：总骑行量（目标变量）

小时数据（hour.csv）
与day.csv相似，额外包含hr字段表示小时（0-23）

模型说明
本项目使用以下三种机器学习算法进行预测：

线性回归（Linear Regression）
原理：假设目标变量与特征之间存在线性关系
优点：模型简单，可解释性强，计算速度快
适用场景：特征与目标变量呈近似线性关系时表现最优

决策树（Decision Tree）
原理：通过递归分割特征空间构建树形决策模型
优点：可处理非线性关系，无需特征标准化
超参数：max_depth=10，限制树的深度防止过拟合
适用场景：数据有复杂的分段线性特性时

支持向量机（Support Vector Regression）
原理：在高维空间中寻找最优回归超平面
优点：对高维数据处理能力强，泛化性能好
超参数：kernel=RBF高斯核，C=100，gamma=0.01
适用场景：非线性复杂关系的预测任务

性能评估指标
MSE（均方误差）：预测误差平方的平均值，值越小越好
R²（决定系数）：0-1之间，越接近1模型越好，表示模型解释方差的比例
