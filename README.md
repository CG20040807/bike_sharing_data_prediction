# 🚲 共享单车骑行量预测

> 用数据读懂城市出行，用模型预测骑行需求。

本项目基于 **Python** 构建，围绕共享单车骑行数据展开探索性分析与机器学习预测。  
通过天气、温度、季节、时间等因素，分析骑行量变化规律，并尝试建立预测模型，为城市交通调度与资源配置提供参考。

---

## ✨ 项目亮点

- 📊 **数据分析**：从季节、天气、温度、工作日等维度观察骑行规律
- 📈 **可视化展示**：使用 `Matplotlib` / `Seaborn` 绘制多种统计图
- 🤖 **机器学习建模**：对骑行量进行回归预测
- 📉 **模型评估**：使用 `MSE`、`R²` 对不同模型进行比较
- 🧠 **业务洞察**：帮助理解共享单车需求的影响因素

## 🤖 模型说明

本项目使用了以下回归模型：

- `LinearRegression`：线性回归
- `DecisionTreeRegressor`：决策树回归
- `SVR`：支持向量回归

评价指标如下：

- **MSE**：均方误差，越小越好
- **R²**：决定系数，越接近 1 越好

通过对比不同模型在训练集与测试集上的表现，选择更适合该任务的预测方法。

## 📌 数据结论

从图表中可以观察到几个较明显的现象：

- 🌡 **温度越高，骑行量整体越高**
- 🌤 **天气越好，骑行需求通常越大**
- 🗓 **季节变化会影响骑行量分布**
- ⏰ **通勤时段骑行量更集中**

这些结果说明，共享单车需求具有明显的时间性与环境依赖性。
